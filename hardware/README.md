# Hardware Layer — Smart Safety Belt Firmware

This folder contains the **ESP32 firmware** that turns a physical belt into a
live node in the Smart Mining Safety Belt system. It's provided as a bonus
layer on top of the software demo — the web app works fully with the built-in
simulator even without any hardware connected.

## Wiring Summary

See pin definitions at the top of `esp32_safety_belt/esp32_safety_belt.ino`.
Analog sensors (gas, battery) go to ESP32 ADC pins; MPU6050 and MAX30100
share the I2C bus (SDA/SCL); GPS uses a hardware UART; the SOS button uses a
digital input pin with internal pull-up.

## Flashing

1. Open `esp32_safety_belt/esp32_safety_belt.ino` in the Arduino IDE (or PlatformIO).
2. Install libraries: `ArduinoJson`, `Adafruit_MPU6050`, `Adafruit Unified Sensor`, `MAX30100_PulseOximeter`, `TinyGPSPlus`.
3. Update `WIFI_SSID`, `WIFI_PASSWORD`, and `SERVER_INGEST_URL` / `SERVER_SOS_URL` to point at your backend's LAN IP (not `localhost` — the ESP32 is a separate device).
4. Set `BELT_ID` to match a `beltId` seeded in the database (e.g. `BELT-A1`).
5. Flash to the board and open the Serial Monitor at 115200 baud to confirm WiFi connection and successful POSTs.

## Demo-day tip

If your hardware isn't fully wired up in time, **leave `SIMULATION_ENABLED=true`**
in the backend `.env`. The dashboard will show fully live, realistic data
generated by the simulation engine, using the exact same alert pipeline and
data model the real firmware feeds into. You can demo the whole system end
to end and simply say "here's the real firmware we built for the physical
belt" while showing this code and the wired prototype.
